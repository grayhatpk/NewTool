Docker API

When you install docker on a host machine, the daemon and CLI communicate with each other through Unix Socket that represents a loopback address. If you want to access the docker application externally, then bind the API over a TCP port. The time you allow the docker API to be accessed over TCP connection through ports such as 2375,2376, 2377 that means a docker CLI which is running outside the host machine will be able to access the docker daemon remotely.

Check if the docker daemon port 2375 is open. 

Nmap -A -p 2375 -Pn 127.0.0.1

4. Fast privilege escalation

There are several methods to escalate privileges, escape the Docker container, and gain control of the Docker host. I'll discuss my preferred approach: mounting the Docker host file system into the container.

Ensure that an Ubuntu or Alpine Docker image is present on the target system. If not, Docker will automatically pull and run it. Note the specific image tag, as Docker pulls and runs the latest image when no tag is specified in the run command.

On our target host, an Ubuntu image with the 18.04 tag is available.

docker run -it -v /:/mnt/ ubuntu:18.04 chroot /mnt/ bash

This command runs the Ubuntu container in interactive mode, mounts the Docker host file system to the /mnt directory, and makes it the root of the filesystem. As a result, the /mnt directory becomes the root (/) file system.

Checking if we are in the privileged container or not. 

cat /proc/1/cgroup     
ps aux
  • We see the word "docker" in the output along with a small number of processes, confirming we are in a container. In contrast, a host system would display a large list of processes, indicating we have escaped the container.
